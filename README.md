# DLP Analyzer

Инструмент для анализа скриншотов рабочих станций и выявления признаков утечки данных и внутренних рисков. Помогает специалисту по экономической/информационной безопасности при первичном разборе DLP-инцидентов: автоматически распознаёт текст на кадрах и помечает те, где есть признаки конфиденциальной информации или подозрительной активности.

> ⚠️ Учебный pet-проект. Для тестов используются только синтетические скриншоты, не содержащие реальных корпоративных данных.

## Что это

Один файл — `dlp-analyzer-browser.html`. Скачайте его и откройте двойным кликом в браузере (Chrome/Edge). Установка не требуется: распознавание текста (OCR, русский + английский) и проверка правил выполняются прямо в браузере, файлы никуда не отправляются.

При первом запуске один раз подгружается движок распознавания (нужен интернет), дальше анализ работает локально.

## Как работает анализ

1. **OCR** — на каждом скриншоте распознаётся текст (русский + английский одновременно). Перед распознаванием кадр проходит предобработку: тёмные интерфейсы осветляются, широкие двухмониторные кадры режутся пополам, мелкий текст увеличивается, а «мусор» от фотообоев отсекается по уверенности распознавания.
2. **Проверка по словарю** — распознанный текст прогоняется через набор правил (регулярные выражения + контрольные суммы). Каждое срабатывание — это «находка» с типом и уровнем критичности.
3. **Скоринг и группировка** — находки собираются по файлам и сотрудникам, считается риск-оценка, формируется отчёт.

Инструмент только **помечает** кадры — итоговое решение принимает аналитик. Из списка инцидентов можно открыть исходный скриншот (с масштабированием) и убедиться в выявленном.

## Что именно ищет (словарь детекции)

Словарь построен по принципу «уровень критичности → категория». Каждую категорию можно включить или отключить прямо в интерфейсе.

### 🔴 Критический уровень — требует немедленной реакции

**Платёжные и банковские реквизиты**
- **Банковская карта** — 13–19 цифр с проверкой по **алгоритму Луна** (отсекает номера заказов, трек-номера и прочие случайные последовательности цифр)
- **Номер счёта** (20 цифр с контекстом «счёт / р/с / расчётный»)
- **БИК банка** (9 цифр)
- **Криптовалютный кошелёк** (Bitcoin, Ethereum)

**Персональные данные**
- **Паспорт РФ** (серия + номер, с усилением по контексту «паспорт / серия»)
- **СНИЛС** — с проверкой **контрольной суммы**
- **Полис ОМС / ДМС** (16 цифр)

**Секреты и доступы**
- **Пароль / секрет** (`password=`, `пароль:`, `token=` и пр.)
- **API-ключ / токен авторизации** (`api_key`, `bearer`, `access_token`…)
- **Приватный SSH / PGP ключ** (заголовок `BEGIN PRIVATE KEY`)

**Экономические и уголовные риски** (лексика)
- **Коррупция / взятка** — взятка, откат, подкуп, «занос», незаконное вознаграждение и пр.
- **Обналичивание / отмывание** — обнал, отмывание, чёрная касса, серая зарплата, однодневка, транзитная схема, дроппер и пр.
- **Криминальная активность** — мошенничество, хищение, вымогательство, шантаж, рейдерство, «слить базу», «пробить по базе» и пр.
- **Уголовный / правовой риск** — уголовное дело, ст. … УК, обыск, следователь, «под следствием» и пр.
- **Слив базы / клиентов** — «выгрузить базу», «база клиентов», «экспорт CRM», `SELECT * FROM`, дамп базы и пр.
- **Тендерные риски** — «заточить ТЗ», «ТЗ под подрядчика», откат по закупке, 44-ФЗ / 223-ФЗ и пр.

### 🟠 Средний уровень — важно, требует проверки

- **ИНН** (10/12 цифр) — с проверкой **контрольной суммы**
- **ОГРН / КПП**
- **SWIFT / корр. счёт**
- **Телефон РФ**
- **Дата рождения**
- **Email (личный)** — gmail, mail.ru, yandex и др. (с допуском OCR-искажений)
- **Гриф секретности** — «конфиденциально», «коммерческая тайна», «ДСП», confidential, top secret и пр.
- **Логин / учётная запись**
- **Номер договора / контракта**
- **Внутренний IP-адрес**, **MAC-адрес**
- **Медицинские данные** — диагноз, МКБ, больничный, история болезни и пр.
- **Водительское удостоверение**, **Загранпаспорт**
- **Финансовый документ** — платёжное поручение, счёт на оплату, акт сверки, КБК и пр.
- **Поиск работы / увольнение** — hh.ru, резюме, вакансия, собеседование, «увольняюсь» и пр. (индикатор нелояльности)

### 🟢 Низкий уровень — для сведения и мониторинга

- **Email (прочий)** — любые адреса вне списка личных доменов
- **Мессенджер / соцсеть** — Telegram, WhatsApp, VK, Discord, Signal, Viber и пр.
- **Облачное хранилище** — Google Drive, Dropbox, Яндекс.Диск, OneDrive, WeTransfer и пр.
- **Внешний носитель** — USB, флешка, съёмный диск

## Точность и защита от обхода

- **Контрольные суммы** (Луна для карт, контрольные цифры ИНН и СНИЛС) — резко снижают ложные срабатывания на случайных числах.
- **Деобфускация** — ключевые категории распознают попытки обхода и типичные ошибки OCR: `0ткат` (ноль вместо «о»), `к0ррупция`, латиница вперемешку с кириллицей (`оtкat`).
- **Взвешенный скоринг** — если на одном кадре набирается несколько находок среднего уровня (≥3), они совокупно эскалируются до критического: несколько слабых сигналов вместе часто важнее одного.
- **Фильтр OCR-шума** — слова с низкой уверенностью распознавания (фотообои, узоры, фон) отбрасываются, чтобы не засорять результаты.

## Свой словарь и белый список

Словарь расширяется под конкретное расследование прямо в интерфейсе, без правки кода:

- **Свои ключевые слова** — названия проектов, фамилии контрагентов, кодовые слова. Ищутся **по основе слова** (например, «увольн» покрывает «увольнение», «уволился»).
- **Белый список** — подстроки, которые никогда не считать нарушением (корпоративный домен почты, свои служебные номера) — убирает постоянный шум.
- **Импорт / экспорт словаря в JSON** — можно вести общий словарь отдела и обмениваться им.

Свой словарь и белый список сохраняются в браузере между сессиями.

## Возможности интерфейса

- Загрузка скриншотов перетаскиванием, выбором файлов или целой папки
- Режим «Следить за папкой» (Chrome): новые скриншоты в папке распознаются автоматически, по завершении в папку сохраняется итоговый отчёт
- Имя сотрудника определяется по структуре папок: `HS/<Сотрудник>/<дата>/скрины`
- Управление анализом: запуск / пауза / стоп с продолжением ровно с того же места
- Дашборд по сотрудникам с риск-оценкой и разбивкой по типам нарушений
- Таблица результатов с двухуровневой группировкой (сотрудник → папка со скринами), сворачиваемыми группами, полным распознанным текстом, поиском и пагинацией
- Просмотр исходного скриншота с масштабированием (зум колесом, 1:1, по ширине, перетаскивание)
- Экспорт результатов в **CSV** и в **Excel (.xlsx)** с настоящей группировкой строк (сворачиваемая «Структура»)

## Приватность

Вся обработка локальная: скриншоты, распознанный текст и результаты не покидают компьютер. Единственное сетевое обращение — одноразовая загрузка OCR-движка с CDN при первом запуске.

## Дисклеймер

Проект разрабатывался как личный тренировочный инструмент в рамках развития компетенций на стыке экономической безопасности и анализа данных. Не предназначен для production-использования и не связан с какой-либо корпоративной DLP-системой работодателя.
