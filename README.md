# DLP Analyzer

Инструмент для анализа скриншотов рабочих станций и выявления признаков утечки данных. Прототип в помощь специалисту по экономической/информационной безопасности при разборе инцидентов с DLP-системами.

> ⚠️ Учебный pet-проект. Для тестов используются только синтетические скриншоты, не содержащие реальных корпоративных данных.

## Что решает

При расследовании DLP-инцидентов аналитик часто получает большой массив скриншотов рабочих станций сотрудников. Ручной просмотр каждого кадра — медленно. Этот инструмент автоматизирует первичную фильтрацию: размечает кадры, на которых с высокой вероятностью присутствуют признаки конфиденциальной информации, и группирует их по категориям рисков.

## Возможности

- Пакетная загрузка и анализ скриншотов
- Автоматический анализ: новые скрины в папке `input/` распознаются и проверяются сразу, без ручного запуска
- Привязка результатов к сотруднику: задаётся имя сотрудника (или раскладка по подпапкам `input/<Сотрудник>/`), и все находки группируются по нему, а не в общий список
- Выделение текста на изображениях (OCR)
- Поиск признаков рисков по ключевым категориям (персональные данные, финансовая информация, признаки переписки с внешними адресатами и др.)
- Веб-интерфейс для просмотра результатов и приоритизации кадров
- Поиск по распознанному тексту скринов (в т.ч. по нескольким вариантам слова через запятую)
- Экспорт результатов анализа в CSV и JSON (что и на каком скрине найдено)
- Локальная обработка — данные не покидают рабочую станцию аналитика

## Стек

- Python (backend, обработка изображений и OCR)
- HTML/JS (frontend)
- Bash (установочные скрипты)

## Запуск

**macOS / Linux:**

```bash
./install.sh        # установка зависимостей
./run.sh            # запуск приложения
```

**Windows:**

Сначала установите [Tesseract OCR для Windows](https://github.com/UB-Mannheim/tesseract/wiki)
(при установке отметьте русский языковой пакет), затем:

```bat
install.bat         :: установка зависимостей
run.bat             :: запуск приложения (браузер откроется автоматически)
```

После запуска дашборд доступен по адресу http://127.0.0.1:8000.
Скриншоты для анализа кладите в папку `input/`.

## Структура

```
backend/      — обработка изображений, OCR, классификация
frontend/     — веб-интерфейс для просмотра результатов
install.sh    — установка зависимостей
run.sh        — запуск приложения
```

## Дисклеймер

Проект разрабатывался как личный тренировочный инструмент в рамках развития компетенций на стыке экономической безопасности и анализа данных. Не предназначен для production-использования и не связан с какой-либо корпоративной DLP-системой работодателя.
